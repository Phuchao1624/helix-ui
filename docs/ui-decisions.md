# Helix / Hope School — UI decisions

## Direction

Hope School should feel like a calm, credible learning environment: editorial hierarchy, clear academic utility and a distinct Helix rhythm. The product is not a generic dashboard of equal cards.

## System choices

- **Colour:** navy carries focus and primary actions; Hope blue/cyan carries navigation and information; green confirms progress; a small warm accent marks time-sensitive detail. Colour never substitutes for text status.
- **Grid:** wide operational pages use an intentional 12-column rhythm; reading and forms remain narrower. Desktop has one dominant action with supporting information. Mobile changes the order so the next action remains in the first viewport, instead of merely stacking desktop cards.
- **Surfaces:** use panels only when they group a decision or task. Avoid a card inside a card and avoid decorative shadows as structure.
- **Type:** large, compact display headlines establish orientation; small uppercase metadata gives context; body copy stays short and concrete.
- **Helix identity:** three vertical bars appear as a restrained progress/identity marker rather than as a repeated decoration.
- **Motion:** interaction feedback should be brief and purposeful (100–400ms for press, progress and tab state), and honour reduced motion. No autoplay, parallax or decorative animation is required for learning tasks.
- **Components:** retain the shared buttons, search, status badges, progress, focus states and navigation behaviour. New page composition may layer these primitives differently.

## Patterns intentionally avoided

- Equal-height grids of generic course cards.
- Oversized welcome copy that pushes the first action below the fold.
- KPI tiles presented before the learner knows what to do next.
- Redundant controls or CTAs that navigate to the same place without explaining why.
- Placeholder images used only as visual filler.

## Inspiration translated, not copied

The visual language borrows the clarity of contemporary editorial education products: one strong next-step panel, quiet metadata, asymmetric support bands and visible learning momentum. It does not copy another product’s brand, artwork or layout verbatim.
